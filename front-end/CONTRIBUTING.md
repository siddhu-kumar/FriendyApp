🔹 Core Change Types

feature
A new functionality added to the project.
Example: Add user login system

enhancement
An improvement to an existing feature (better UX, performance, etc.).
Example: Improve dashboard loading speed

update
A general modification that doesn’t clearly fit feature or fix (e.g., minor logic tweaks).
Example: Update API response handling

change
A broad or mixed modification affecting multiple areas without a single clear purpose.
Use sparingly — “update” is usually better.

🔹 Fixing & Stability

fix
A bug fix.
Example: Fix login crash on invalid input

hotfix
A critical fix applied urgently, usually in production.
Example: Hotfix payment failure issue

🔹 Code Maintenance

refactor
Code changes that improve structure without changing behavior.
Example: Refactor authentication logic

cleanup
Removing unnecessary code, files, or comments.
Example: Cleanup unused imports

chore
Routine tasks that don’t affect application logic.
Example: Update dependencies

🔹 Versioning & Releases

version
Used when bumping or tagging versions.
Example: Version 2.1.0 release

release
Marks a deployment or packaged release.
Example: Release production build

🔹 Adding Things

add
Adding files, configs, or small pieces of functionality (lighter than feature).
Example: Add README file

🔹 Documentation & Config

docs
Documentation changes only.
Example: Update API docs

config
Configuration-related updates.
Example: Update ESLint rules

🔹 Testing & CI/CD

test
Adding or modifying tests.
Example: Add unit tests for auth

ci
Continuous integration / deployment changes.
Example: Update GitHub Actions workflow
